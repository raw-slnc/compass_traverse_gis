Compass Traverse GIS Manual

QGIS プラグイン Compass Traverse GIS の基本操作と、野帳・計算・レイヤ管理の考え方をまとめた簡易マニュアルです。

A quick reference for Compass Traverse GIS — field notebook entry, traverse computation, layer management, and export.

1. 概要

このプラグインは、コンパス測量の野帳入力、接続点・閉合点管理、トラバース計算、レイヤ生成、帳票エクスポートを一体で扱うための作業画面です。

レイヤはプラグイン管理レイヤとして生成されます。必要に応じてシンボロジ変更はできますが、元データ編集は野帳テーブル側で行ってください。

2. 基本構成

左パネルデータ状態、プロジェクト・作業名選択、ブロック名称、成果物管理
野帳 タブ野帳入力、測量設定、接続点・閉合点、位置入力
Project タブプロジェクト管理情報、計算サマリー、言語切替

3. データ状態グループ

左パネル上部の「データ状態」グループには、作業データの保存状況が表示されます。

現行データ現在読み込まれている SQLite ファイル名。QGIS プロジェクト未保存時は「データ未作成」と表示されます。
切替退避済みデータが存在する場合に有効になります。過去データを「保存データを切替」ダイアログから読み込み直せます。
新規作成現行データをアーカイブフォルダへ退避し、新しい SQLite を作成します。QGIS プロジェクトを保存してから実行してください。
SQLite は QGIS プロジェクトと同じフォルダに作成されます。退避済みファイルは _compass_traverse_gis_archive フォルダに保存されます。

4. 野帳入力

  • 測量点（測量点）と目標点（目標点）は、数値に限らず自由な文字列で入力できます。
  • 接続点 と 閉合点 は、ダブルクリックで測点候補から選択できます。
  • 位置 列はダブルクリックで緯度経度入力モーダルを開きます。
  • 空行は視認上の区切りとして使用できます。

5. 野帳インポート

測量テーブル読込 を押すと、CSV または Excel 形式の測量表を野帳へ読み込みます。

  • 対応形式は .csv, .xlsx, .xls, .xlsm です。
  • CSV は utf-8-sig, cp932, utf-8 を順に試して読み込みます。
  • 区切り文字は自動判定され、判定できない場合はカンマ区切りとして扱います。

自動検知される列名

読込時はヘッダ名を見て、対応列の候補を自動検知します。空白、記号、大小文字の差はある程度吸収されます。

野帳列自動検知キーワード例
測量点測量点, 測点, 起点, from
目標点目標点, 目標, 到達点, target, to
接続点接続点, 接続
閉合点閉合点, 閉合
SDSD, 斜距離, 射距離, 実測距離
INCINC, 高低, 高低角, 鉛直角
AZAZ, 方角, 方位角, 方位, 方向角
HDHD, 水平距離
緯度緯度, lat, latitude
経度経度, lon, lng, longitude

読込時の流れ

  1. ファイルを選択する
  2. 列対応ダイアログで、自動検知結果を確認する
  3. 必要なら手動で対応列を変更する
  4. 「測量点」と「目標点」に相当する列が設定されていれば読み込む
必須列: 少なくとも「測量点」と「目標点」の対応が必要です。これらが未設定のままでは読み込みできません。

緯度・経度列を対応させた場合は、野帳の「位置」列に内部データとして保存され、セル表示は「設定済」になります。

読み込み後は野帳テーブルがその内容で置き換わります。必要に応じて レイヤ更新 を実行して計算とレイヤへ反映してください。

6. 位置と開始座標

  • 「位置」が入力された行は、開始座標逆算のアンカーとして使われます。
  • 同じ連結ブロック内に位置が複数ある場合は、若い行番号側を優先します。
  • 開始座標が確定できない場合、計算やレイヤ更新は停止します。

7. 計算・レイヤ更新

レイヤ更新 を押すと、現在の野帳内容から計算を行い、管理レイヤを再生成します。

  • HD、ΔX、ΔY を更新
  • 点・線・面レイヤを生成
  • 連携先がある場合は、連携先作業の完了データも同時表示

8. エクスポート

  • エクスポート は完了フラグ（完了フラグ = 1）が立っている作業のみ実行できます。
  • HTML 帳票、地図 PDF、関連出力をまとめて生成します。
  • 出力先フォルダはエクスポートダイアログで指定します。

9. 運用メモ

  • 測点名を変更した場合、参照している接続点・閉合点は自動追従します。
  • 連携は作業単位です。連携先は完了済みデータである必要があります。
  • どうしてもレイヤを自由編集したい場合は、別レイヤへ複製して使用してください。
  • プラグインを隠す ボタンでプラグインを非表示にし、QGISキャンバスを広く使えます。ツールバーアイコンで再表示できます。

1. Overview

This plugin provides an integrated workspace for compass survey entry, connecting-point and closure management, traverse computation, layer generation, and report export.

Layers are managed by the plugin. You may change symbology, but edit source data in the field notebook table, not in the QGIS layer.

2. Layout

Left panelWorkspace Status, Project / Work Name selection, Block Name, Deliverables
Notebook tabField notebook entry, Survey Settings, Connect To / Close To, Geo coordinates
Project tabProject management info, Calculation Summary, language toggle

3. Workspace Status Group

The Workspace Status group at the top of the left panel shows the current data storage state.

Current DataThe SQLite file currently loaded. Shows "Not Created" when the QGIS project has not been saved yet.
SwitchEnabled when archived data exists. Use the "Switch Saved Data" dialog to reload a past dataset.
NewArchives the current SQLite to the archive folder and creates a new one. Save the QGIS project first.
The SQLite file is created in the same folder as the QGIS project. Archived files are stored in _compass_traverse_gis_archive.

4. Field Notebook Entry

  • From and To accept any string, not just numbers.
  • Connect To and Close To: double-click to pick from a station list.
  • Geo column: double-click to open the latitude/longitude entry dialog.
  • Blank rows are allowed as visual separators.

5. Import Survey Table

Click Import Survey Table to load a CSV or Excel file into the notebook.

  • Supported formats: .csv, .xlsx, .xls, .xlsm.
  • CSV encoding is tried in order: utf-8-sig, cp932, utf-8.
  • Delimiter is auto-detected; falls back to comma.

Import workflow

  1. Select a file
  2. Review auto-detected column mappings in the dialog
  3. Adjust manually if needed
  4. Import proceeds when at least From and To are mapped
Required: At least the From and To columns must be mapped. Import is blocked otherwise.

If latitude/longitude columns are mapped, the data is stored internally in the Geo column and the cell shows "Set".

The notebook table is replaced on import. Run Update Layers to apply the changes to the calculation and layers.

6. Geo and Start Coordinate

  • A row with a Geo value is used as an anchor to back-calculate the start coordinate.
  • If multiple Geo values exist in the same block, the earliest row takes priority.
  • If the start coordinate cannot be resolved, computation and layer update will stop.

7. Update Layers

Click Update Layers to run the traverse computation and regenerate managed layers.

  • Updates HD, dX, and dY
  • Generates point, line, and polygon layers
  • If a linked project exists, its completed data is displayed simultaneously

8. Export

  • Export is only available when Completion flag is set (Completion = 1).
  • Generates HTML report, map PDF, and related outputs together.
  • The output folder is specified in the Export dialog.

9. Notes

  • When a station name is changed, Connect To and Close To references update automatically.
  • Linking works per work record. The linked target must be marked complete.
  • If you need to freely edit a layer, duplicate it to a separate layer first.
  • The Hide Plugin button hides the plugin panel so you can use the full QGIS canvas. Click the toolbar icon to restore it.
